# gotosocial-comment-embed

A unified comment section for l3on.site combining GoToSocial (Fediverse) replies and Webmentions. Runs entirely on a single Cloudflare Worker — no client-side config needed in the blog.

## How it works

```
BearBlog footer loads:
  https://gts-comment-worker.l3on.workers.dev/l3on-comments.js

l3on-comments.js reads the page, builds a comments section, and appends it to <main>.
It fetches both Fediverse comments and Webmentions from the worker API.
```

The worker serves three routes:

| Route | What it does |
|---|---|
| `/l3on-comments.js` | The embed script (built from `packages/embed`) |
| `/fediverse/:statusId` | Proxies GoToSocial API, filters public replies, caches 5 min |
| `/webmentions?target=...&also=...` | Proxies webmention.io, caches 5 min |

## Project layout

```
packages/
  embed/
    src/l3on-comments.ts   — main embed script entry
    src/fediverse.ts       — Fediverse fetch/render logic
    src/webmention.ts      — Webmention fetch/render logic
    src/shared.ts          — shared sanitizing/render helpers
    dist/                  — built output (gitignored, generated by build)
    demo.html              — local test page with fake data controls
  worker/
    src/index.ts           — Cloudflare Worker (serves routes above)
    wrangler.toml          — worker config, allowed origins, cache TTL
```

## Development

**Prerequisites:** Bun, Node, Wrangler (`npm i -g wrangler`)

```bash
# Install dependencies
bun install

# Build embed script → dist/
bun run build

# Local worker dev (runs on http://127.0.0.1:8787)
bun run dev:worker

# Test locally — open in browser, no server needed
packages/embed/demo.html
```

The demo page has radio controls to simulate fedi and webmention states (empty / with content / error) without deploying.

For local worker dev you need a `.dev.vars` file in `packages/worker/`:

```
GTS_INSTANCE_URL=https://rex.cat
GTS_ACCESS_TOKEN=your_token_here
```

## Deploy

```bash
bun run build    # always build first
bun run deploy   # deploys worker to Cloudflare
```

The worker bundles `dist/l3on-comments.js` as a text import, so `bun run build` must run before `deploy`.

## BearBlog setup

**Footer** — one script tag, nothing else needed:
```html
<script src="https://gts-comment-worker.l3on.workers.dev/l3on-comments.js"></script>
```

**Per post/page** — put a hidden span anywhere in the article body to enable fedi comments:
```html
<span data-fedi-id="YOUR_GTS_STATUS_ID" hidden></span>
```

Without this span, the page still shows the webmention section but no fedi comments.

**Webmention alias URLs** — if you renamed a page and want to pull in webmentions from the old URL:
```html
<span data-wm-also="https://l3on.site/old-url/" hidden></span>
```

Multiple old URLs: add multiple spans. Trailing slashes are normalized automatically.

## Where CSS lives

All styles are in BearBlog **Custom CSS** — nothing is injected by the script. Edit there to change appearance. The script uses these CSS classes:

- `.l3on-comments-section`, `.l3on-comments-group`, `.l3on-comments-label` — section layout
- `.l3on-comment`, `.l3on-comment-avatar`, `.l3on-comment-body` — comment cards
- `.l3on-comment-meta`, `.l3on-comment-handle`, `.l3on-comment-badge` — author line
- `.l3on-comment-info`, `.l3on-comment-time`, `.l3on-comment-stats` — timestamp/stats
- `.l3on-comments-header`, `.l3on-comment-stats-header`, `.l3on-stats-item` — count header
- `.l3on-help`, `.l3on-help-form`, `.l3on-help-input`, `.l3on-help-btn` — interaction toggles
- `.l3on-loading`, `.l3on-empty`, `.l3on-error` — states
- `#webmentions` — webmention container (also styled by Custom CSS)

## Worker secrets (Cloudflare dashboard)

| Variable | Where to set | Notes |
|---|---|---|
| `GTS_ACCESS_TOKEN` | Cloudflare secret | GoToSocial API token |
| `GTS_INSTANCE_URL` | `wrangler.toml` vars | e.g. `https://rex.cat` |
| `ALLOWED_ORIGINS` | `wrangler.toml` vars | comma-separated, e.g. `https://l3on.site,https://lok.bearblog.dev` |
| `CACHE_TTL` | `wrangler.toml` vars | seconds, default `300` |

To rotate the token: `npx wrangler secret put GTS_ACCESS_TOKEN`

## Excluded paths

The comment section is suppressed on these paths (defined in `l3on-comments.ts`):

```typescript
const EXCLUDED_PATHS = ['/journal', '/guestbook'];
```

Edit that constant and redeploy to change.
